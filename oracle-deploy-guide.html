Deploy Skin Analysis App to Oracle Cloud

Complete step-by-step guide to deploy your FastAPI + YOLO skin analysis application to Oracle Cloud Infrastructure. Includes Python environment setup, model deployment, and production configuration.

1

Prerequisites

What You Need

✓ SSH Client (Windows: PowerShell/PuTTY)

Configure Security List

Networking → Virtual Cloud Networks → [Your VCN] → Security Lists → Add Ingress Rules

Add Ingress Rule for Port 8000

+ Add FastAPI Port (8000)

  • Source Type: CIDR
  • Source CIDR: 0.0.0.0/0
  • IP Protocol: TCP
  • Destination Port: 8000
  • Description: FastAPI Skin Analysis App

Connect via SSH

Get Your Instance Public IP

Compute → Instances → [Your Instance] → Public IP Address

Windows PowerShell Commands

PowerShell 
# Step 1: Navigate to your SSH key directory
cd C:\orc\skin-app

# Step 2: Fix SSH key permissions (required on Windows)
icacls "ssh_prv.key" /inheritance:d /grant:r "$($env:USERNAME):(F)"

# Step 3: Connect to your instance
ssh -o IdentitiesOnly=yes -i ssh_prv.key opc@YOUR_PUBLIC_IP

Connection Details

Username opc
Port 22
Key File ssh_prv.key
Sudo Access Yes

Setup Python Environment

Create Virtual Environment & Install Dependencies

Bash (on Oracle Linux) 
# Navigate to project directory
cd /home/opc/skin-analysis

# Create virtual environment
python3.11 -m venv venv

# Activate virtual environment
source venv/bin/activate

# Upgrade pip
pip install --upgrade pip

# Install project dependencies
pip install -r requirements.txt

# Verify installation
pip list | grep -E "fastapi|uvicorn|ultralytics"

Low Memory Warning

If using Free Tier (1GB RAM), installation may be slow. Consider adding swap space:

sudo dd if=/dev/zero of=/swapfile bs=1M count=2048
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

Configure Instance Firewall

Two Firewall Layers

Oracle Cloud has two firewall layers: Security List (cloud level - done in Step 4) and firewalld (OS-level - do now).

Open Port 8000 in firewalld

Bash (on Oracle Linux) 
# Check firewall status
sudo systemctl status firewalld

# Open port 8000 permanently
sudo firewall-cmd --permanent --add-port=8000/tcp

# Optional: Open ports 80 and 443 for future use
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp

# Reload firewall rules
sudo firewall-cmd --reload

# Verify ports are open
sudo firewall-cmd --list-ports

Expected Output

8000/tcp 80/tcp 443/tcp

Create Systemd Service (Production)

Why Systemd?

Systemd ensures your app starts automatically on boot and restarts if it crashes.

Create Service File

Bash (on Oracle Linux) 
# Create systemd service file
sudo nano /etc/systemd/system/skin-analysis.service

Service File Content

skin-analysis.service 
[Unit]
Description=Skin Analysis FastAPI Application
After=network.target

[Service]
Type=simple
User=opc
Group=opc
WorkingDirectory=/home/opc/skin-analysis
Environment="PATH=/home/opc/skin-analysis/venv/bin"
ExecStart=/home/opc/skin-analysis/venv/bin/python app.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

Enable and Start Service

# Reload systemd
sudo systemctl daemon-reload

# Enable service to start on boot
sudo systemctl enable skin-analysis

# Start the service
sudo systemctl start skin-analysis

# Check status
sudo systemctl status skin-analysis

# View logs
sudo journalctl -u skin-analysis -f

Troubleshooting

Can't access the app from browser?

  • • Check Security List has port 8000 open (Step 4)
  • • Verify firewalld has port 8000 open: sudo firewall-cmd --list-ports
  • • Confirm app is running: sudo ss -tlnp | grep 8000
  • • Check app logs: sudo journalctl -u skin-analysis -n 50

Models not loading / Out of memory?

  • • Check available memory: free -h
  • • Add swap space (see Step 8)
  • • Consider upgrading to a larger instance shape
  • • Load fewer models or use smaller model variants

OpenCV / libGL errors?

  • • Install missing libraries: sudo dnf install -y mesa-libGL
  • • Use headless OpenCV: pip install opencv-python-headless

Service won't start?

  • • Check service status: sudo systemctl status skin-analysis
  • • View detailed logs: sudo journalctl -u skin-analysis -e
  • • Verify paths in service file are correct
  • • Test running manually first before using systemd

Nginx Reverse Proxy (Optional)

Bash (on Oracle Linux) 
# Install Nginx
sudo dnf install -y nginx

# Create config
sudo nano /etc/nginx/conf.d/skin-analysis.conf

# Add this configuration:
server {
    listen 80;
    server_name YOUR_DOMAIN_OR_IP;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

# Start Nginx
sudo systemctl enable nginx
sudo systemctl start nginx
